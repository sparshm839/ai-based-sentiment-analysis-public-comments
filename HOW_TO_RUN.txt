How To Run The Sentiment Analysis Project
=========================================

Open a terminal in the project root folder first.
All paths below are relative to the project root.


1. Install backend requirements
-------------------------------
Run:

    cd backend
    pip install -r requirements.txt

If your system uses the Python launcher on Windows, you can use:

    py -m pip install -r requirements.txt


2. Start the backend
--------------------
From the backend directory, run:

    python -m uvicorn api:app --host 127.0.0.1 --port 8000

Or on Windows with the Python launcher:

    py -m uvicorn api:app --host 127.0.0.1 --port 8000

The backend should run at:

    http://127.0.0.1:8000

Health check:

    http://127.0.0.1:8000/health


3. Install frontend requirements
--------------------------------
Open a second terminal in the project root folder.

Run:

    cd frontend/clarity-sentiment-analyzer-main
    npm install


4. Start the frontend
---------------------
From frontend/clarity-sentiment-analyzer-main, run:

    npm run dev

The frontend should run at:

    http://localhost:8080

or:

    http://127.0.0.1:8080


5. Using the transformer model option
-------------------------------------
In the website, choose the Local Transformer model only after the backend is running.
The frontend sends requests to:

    http://127.0.0.1:8000/predict


6. If the model is not locally installed
----------------------------------------
The first backend run may download Hugging Face models automatically.
This can take time and requires internet access.

Models used by the backend:

    cardiffnlp/twitter-roberta-base-sentiment-latest
    nahiar/spam-detection-xlm-roberta-v3

If startup appears stuck, wait for the model download/load to finish.
Large model files are intentionally not included in the zip.

If model download fails:

    1. Check your internet connection.
    2. Run the backend command again.
    3. If Hugging Face rate limits appear, set an HF_TOKEN environment variable.
    4. Make sure there is enough disk space for model cache files.

If you want faster startup after models are downloaded, do not delete the model cache.


7. Useful commands
------------------
Run frontend production build:

    cd frontend/clarity-sentiment-analyzer-main
    npm run build

Run backend only:

    cd backend
    python -m uvicorn api:app --host 127.0.0.1 --port 8000

Run frontend only:

    cd frontend/clarity-sentiment-analyzer-main
    npm run dev


8. What is not included in the clean zip
----------------------------------------
The clean zip excludes:

    node_modules
    dist
    __pycache__
    backend/model_cache
    downloaded model files
    temporary cache folders

These are recreated by installing dependencies or running the project.
